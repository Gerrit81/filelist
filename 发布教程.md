# 📘 手动发布到 GitHub 教程

> 你已经有了 GitHub 账号，下面一步步教你如何把这个项目发布到 GitHub。

---

## 第一步：安装 Git

### Windows

1. 打开 https://git-scm.com/download/win
2. 下载安装包，双击安装
3. 安装过程中全部保持默认选项即可（一路点击 Next）

安装完成后验证：

```powershell
git --version
# 应输出类似: git version 2.47.0.windows.1
```

---

## 第二步：配置 Git 用户信息

打开 PowerShell 或 Git Bash，设置你的 GitHub 用户名和邮箱：

```bash
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub邮箱@example.com"
```

> 📌 这里的用户名和邮箱建议与 GitHub 账号一致，这样提交记录会正确关联到你的 GitHub 头像。

验证配置：

```bash
git config --global user.name
git config --global user.email
```

---

## 第三步：在项目中初始化 Git 仓库

打开 PowerShell，进入项目目录：

```powershell
cd E:\Project\filelist
```

初始化 Git 仓库：

```bash
git init
```

---

## 第四步：创建 .gitignore 文件

有些文件不应该被提交到 GitHub（如缓存、数据库、Session 等）。

创建 `.gitignore` 文件：

```bash
# 在项目根目录创建 .gitignore
```

`.gitignore` 内容如下（复制粘贴）：

```
# 缓存文件
cache/
!cache/.gitkeep

# 缩略图
thumbs/

# Session 数据
session/

# SQLite 数据库
*.db

# 配置文件（包含敏感信息）
config.json

# 系统文件
Thumbs.db
desktop.ini

# 临时文件
*.tmp
*.log
```

也可以用命令一键创建：

在 PowerShell 中执行：

```powershell
@"
# 缓存文件
cache/
!cache/.gitkeep

# 缩略图
thumbs/

# Session 数据
session/

# SQLite 数据库
*.db

# 配置文件（包含敏感信息）
config.json

# 系统文件
Thumbs.db
desktop.ini

# 临时文件
*.tmp
*.log
"@ | Out-File -Encoding utf8 .gitignore
```

---

## 第五步：首次提交代码

### 5.1 查看当前状态

```bash
git status
```

这会列出所有未被跟踪的文件。

### 5.2 将所有文件添加到暂存区

```bash
git add .
```

### 5.3 提交

```bash
git commit -m "初始提交：文件浏览器 v2.0.8"
```

---

## 第六步：在 GitHub 上创建远程仓库

1. 打开浏览器，登录 https://github.com
2. 点击右上角的 **`+`** 号 → **`New repository`**
3. 填写仓库信息：
   - **Repository name**（仓库名）：例如 `file-browser` 或 `filelist`
   - **Description**（描述，可选）：`轻量级 PHP 文件目录浏览系统`
   - **Public** 或 **Private**：选择 Public（公开）或 Private（私有）
   - ⚠ **不要勾选** "Add a README file"（因为我们已经有了）
   - ⚠ **不要勾选** ".gitignore"（因为我们已经创建了）
   - ⚠ **不要勾选** "Choose a license"
4. 点击 **`Create repository`**

---

## 第七步：关联远程仓库并推送

创建仓库后，GitHub 会显示一个页面，里面有命令提示。

找到 **"…or push an existing repository from the command line"** 这一段，复制其命令：

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

> 📌 将 `你的用户名` 和 `仓库名` 替换为实际值。

执行后会弹出 GitHub 登录窗口（或要求输入用户名和密码/Token）。

---

## 第八步：验证

1. 刷新 GitHub 仓库页面
2. 你应该能看到所有代码文件已经上传
3. README.md 会自动渲染为仓库首页的说明文档

---

## 🔮 后续更新代码

当你修改代码后，再次推送更新：

```bash
# 1. 查看修改了哪些文件
git status

# 2. 添加所有修改
git add .

# 3. 提交（写上这次改了什么）
git commit -m "描述本次修改了哪些内容"

# 4. 推送到 GitHub
git push
```

---

## ❓ 常见问题

### Q: 推送时提示 "Authentication failed" 怎么办？

A: GitHub 已不支持密码登录，需要使用 **Personal Access Token**：

1. 登录 GitHub → 右上角头像 → **Settings**
2. 左侧菜单 → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
3. 点击 **Generate new token (classic)**
4. 勾选 `repo` 权限，点击生成
5. **复制生成的 Token**（只显示一次！）
6. 推送时密码栏粘贴这个 Token

### Q: 推送时提示 "failed to push some refs"？

A: 通常是远程仓库已有文件（如 README）。执行：

```bash
git pull origin main --allow-unrelated-histories
git push origin main
```

### Q: 中文文件名为乱码？

A: Git 默认可以正确处理 UTF-8 编码的文件名。确保保存代码文件时使用 UTF-8 编码。

### Q: 如何把某个已跟踪的文件加入 .gitignore？

```bash
git rm --cached 文件名
# 然后编辑 .gitignore 加入该文件，最后提交
```

---

## 📦 总结流程

```
安装 Git → 配置用户信息 → git init → 创建 .gitignore
→ git add . → git commit → GitHub 创建仓库
→ git remote add origin → git push
```

完成以上步骤后，你的项目就成功发布到 GitHub 了！🎉
