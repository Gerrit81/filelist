# 手动更新 GitHub 版本指南（浏览器操作，无需 Git）

## 前置条件

- ✅ 已注册 GitHub，已建仓库，已上传过一版
- 🎯 现在要把旧版本更新到新版本
- 💻 不用 Git 命令行，全靠浏览器操作

---

## 核心问题回答

| 问题 | 答案 |
|------|------|
| **要新建版本号吗？** | 是的，在 Releases 页面创建新版本标签（如 `v2.5.1`） |
| **上传全量还是只改的？** | **建议全量上传**——GitHub 会自动比对，同名文件覆盖，你不需要手动挑文件，也不怕漏传 |

---

## 操作步骤（3 步搞定）

### 第 1 步：上传文件

1. 浏览器打开你的 GitHub 仓库首页
2. 点击 **Add file** 按钮 → **Upload files**
3. 在本地项目文件夹中，全选文件拖进去
4. **⚠️ 注意：以下文件/文件夹不要选**（每次上传都排除）：

   ```
   config.json     ← 删掉，含本地路径
   app.db          ← 删掉，含用户密码
   cache/          ← 删掉，本地缓存
   thumbs/         ← 删掉，本地缩略图
   session/        ← 删掉，会话文件
   backups/        ← 删掉，本地备份
   v-trae/         ← 删掉，本地参考不参与发布
   ```

5. 提交信息写：`v2.5.1 修复中文文件名显示错误`
6. 选择 **Commit directly to the `main` branch**
7. 点击绿色 **Commit changes** 按钮

---

### 第 2 步：创建版本标签（创建 Release）

1. 仓库首页 → 右侧边栏找到 **Releases** → 点击
2. 点击 **Create a new release**
3. 填写内容：

   | 字段 | 填写内容 |
   |------|----------|
   | **Tag version** | `v2.5.1`（点下拉框直接输入新版本号，选 Create new tag） |
   | **Release title** | `v2.5.1` |
   | **Describe** | 把 CHANGELOG.md 里此版本的更新内容粘贴过来 |

4. 点击绿色 **Publish release** 按钮

---

### 第 3 步：验收

发布后 GitHub 会自动生成源码包：

- **Source code (zip)** — 自动打包
- **Source code (tar.gz)** — 同上，tar 格式

点击下载解压，确认文件齐全、目录结构正确，就完成了。

---

## 版本号规则

| 类型 | 版本变化 | 示例 | 触发条件 |
|------|----------|------|----------|
| 小版本 | 末尾 +1 | `v2.5.0` → `v2.5.1` | Bug 修复、样式微调、文案优化 |
| 中版本 | 中间 +1 | `v2.5.0` → `v2.6.0` | 新增功能、新模块、新增配置项 |
| 大版本 | 首位 +1 | `v2.5.0` → `v3.0.0` | 重大架构变更、Breaking Changes |

---

## 小贴士

- 上传文件时 **同名文件会自动覆盖**，放心全选，不用手动挑
- 每次更新都按"上传文件 → 新建 Release"这个流程来就行
- Release 描述写好更新内容，别人下载时就知道改了什么
